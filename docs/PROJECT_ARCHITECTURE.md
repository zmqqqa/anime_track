# Anime Track 项目结构

这个项目是一个动漫记录系统，围绕番剧条目、观看进度、观看历史和用户鉴权展开。

## 目录分工

| 目录 | 说明 |
|---|---|
| `app/` | 页面与 API 路由 |
| `components/` | 仪表盘、番剧列表、详情展示等界面组件 |
| `hooks/` | 动漫数据与观看历史的数据获取逻辑 |
| `lib/` | 数据库访问、鉴权、番剧数据处理、外部元数据抓取 |
| `lib/metadata/` | Bangumi API 对接、AI 元数据生成、合并策略 |
| `database/` | 数据库表结构和迁移脚本 |
| `scripts/` | 部署、备份、批量元数据补全等脚本 |

## 数据表

| 表 | 用途 |
|---|---|
| `users` | 本地登录、注册和角色权限 |
| `anime` | 番剧主数据：标题、状态、进度、评分、简介、标签、封面、首播、声优等 |
| `watch_history` | 按集记录的观看历史，通过 animeId 关联到 anime |

## 数据流

1. 用户在番剧卡片或表单中操作数据
2. 请求进入 `app/api/anime/` 下的路由
3. 路由调用 `lib/anime.ts` 读写数据库
4. 进度增加时，同时写入 `watch_history`
5. 首页和时间线通过 `app/api/history/` 读取观看历史并展示统计

## 外部服务

| 服务 | 用途 |
|---|---|
| NextAuth | 本地账号鉴权 |
| Bangumi API | 番剧元数据查询（封面、集数、评分、声优等） |
| AI（兼容 OpenAI 接口） | 标题标准化、简介生成、标签分类、声优别名翻译 |
